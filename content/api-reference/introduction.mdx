<p>This section is the endpoint-level reference. For the narrative version of how the API fits together, start with <a href="/how-it-works">How it works</a> instead, come back here once you're ready to integrate.</p>

<h2 id="h-base">Base URLs</h2>
<table>
  <tr><th>Environment</th><th>Base URL</th></tr>
  <tr><td>Sandbox</td><td>`https://api.sandbox.nomba-subscriptions.com`</td></tr>
  <tr><td>Production</td><td>`https://api.nomba-subscriptions.com`</td></tr>
</table>
<p>An interactive Swagger reference is also served directly from the API at `/docs` on either environment, useful for trying a request without leaving your browser.</p>

<h2 id="h-format">Response format</h2>
<p>Successful responses are wrapped consistently:</p>

```json
{
  "status": "success",
  "message": "Request successful",
  "data": {}
}
```

<p>Paginated list endpoints return `{ data: [], total: number }` inside `data`.</p>

<h2 id="h-resources">Core resources</h2>
<table>
  <tr><th>Resource</th><th>Endpoints</th></tr>
  <tr><td>Plans</td><td>`POST/GET/PATCH/DELETE /plans`</td></tr>
  <tr><td>Customers</td><td>`POST/GET/PATCH /customers`</td></tr>
  <tr><td>Subscriptions</td><td>`POST/GET /subscriptions`, plus lifecycle actions on `/subscriptions/:id`</td></tr>
  <tr><td>Invoices</td><td>`GET /invoices`, `GET /invoices/:id`</td></tr>
  <tr><td>Payments</td><td>`POST /payments/checkout`, `GET /payments`</td></tr>
  <tr><td>Webhooks</td><td>`POST/GET /webhooks`, `GET /webhooks/:id/deliveries`</td></tr>
  <tr><td>Analytics</td><td>`GET /analytics/*`, see <a href="/merchants/analytics">Analytics</a></td></tr>
</table>
